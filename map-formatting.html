Map Formatting

Need More TC2 Info?

This page focuses on how to format your maps for Typical Colors 2. For class stats, weapons, maps, and other game details, you can also use the community-run Typical Colors 2 Wiki.

Open TC2 Wiki

General

  • Maps follow a strict format in order for them to be properly loaded in the game.
  • You will NOT be able to load your map if this format is not PERFECTLY followed.
  • Some folders are not absolutely necessary for your map to load, however it is highly recommended to include every folder in your map, whether it is being used or not.
  • Folders and models can be used interchangeably—use whichever you prefer!

Tips

  • Keep a copy of your test map—swap in your geometry as you go.
  • Play on test maps to learn sightlines, doors, collisions, etc.
  • Follow the test maps & prop pack while reading this guide.
  • USE THE TEST MAPS!!!

Geometry

  • The Geometry folder contains most of the physical environment that players see and interact with.
  • Disabling CanCollide allows projectiles & players to pass through.
  • Disabling CanTouch makes stickies bounce instead of sticking.
  • Exclude interactables (Doors, kits, KillBricks) & non-blocking parts.

Ignore

  • Contains CameraSpots, KillBricks, NoBuilds, Water.
  • Ignored by hitscan & projectiles (excl. Annihilator’s).
  • CameraSpots: subfolders General, GRN, RED; add NumberValue “FOV”.
  • KillBricks instantly kill on touch.
  • NoBuilds prevent building in bounds.
  • Water → terrain water.

KillBricks without the KillBricks folder

  • You can make a KillBrick that works anywhere in the map hierarchy (inside doors, nested folders, etc.) by using a specific combination of Tags and Attributes.
  • Add these Tags to the part:
    • CharacterTouchRoot
    • Touched
    • trigger_hurt
  • Add these Attributes:
    • ClassName = "trigger_hurt"
    • Damage = kill damage (for example 9001)
    • DamageType = kill icon / damage type name (e.g. "Text")
    • Disabled = false (unchecked)
    • Force = 0, 0, 0 (or a push direction if desired)
    • Scream = true (checked) if you want a death scream
    • Team = team string or leave empty for both teams
    • Text = killfeed text (for example "has died")
    • trigger_hurt = true (checked)
  • This behaves like a normal KillBrick but does not require the part to be under the KillBricks folder. Use alongside existing KillBrick folders/settings to be extra safe.
  • You can also import a ready-made prefab: trigger_hurt.rbxm (place it in Studio and adjust the values you need).

Example Hazard Prefab – Moving Sawblade

  • A fully set up moving sawblade hazard is provided as a prefab: movingsawbladetc2.rbxm.
  • It already uses the KillBricks without the KillBricks folder method internally, so you do not need to weld or attach a separate KillBrick.
  • After inserting it into your map, you normally only change:
    • Its position and rotation in the world.
    • Any cosmetic parts (materials, colors) if you want to restyle it.
  • The prefab uses the DamageType value "FSaw", which is the kill icon used for the sawmill sawblade.
  • The motion is handled purely with constraints and physics (no scripts). If you want to make your own moving hazards, look into:
    • HingeConstraints for rotation.
    • Rod / guide parts to convert rotation into linear motion.
    • For more advanced setups: VectorForce, BodyForce, and AlignPosition (with rigidity) to move players with doors or platforms.

Doors

  • Doors open/close by player input and capture stages.
  • Attributes: Move (studs), MoveTime (seconds).
  • Must contain:
    • PrimaryPart under Pivot—moves when opened; anchored.
    • Trigger with “Properties”:
      • ActivationType (1=touch, 2=melee, 3=projectiles, 4=other).
      • Duration (seconds open).
      • PressMax (-1 for unlimited).
      • Team (“GRN”, “RED”, or blank).
      • WaitForFinish (wait until closed).
    • ForceCloseDuring: stages door locked.
    • ForceOpenDuring: stages door open; no overlap with ForceCloseDuring.

Clips

  • Invisible walls that block players.
  • Ignored by weapons—only blocks players; CanCollide enabled.

Items

  • Resupply cabinets & health/ammo kits.
  • Models match in-game sizes—use them!
  • Parts named “Resupply” for cabinets; “SizeType” for kits (“MediumAmmo”, “SmallHP”).

ControlPoints

  • Maps KOTH, AD, 3CP, Arena.
  • KOTH/Arena: folder “ControlPoint” with hill.
  • AD: ControlPoint1, then ControlPoint2.
  • 3CP: CP1=Red start, CP2=neutral, CP3=Green start.
  • Contains PlayerCaptureTime, PointLocation, CaptureZone.

Briefcases

  • CTF objectives: “Green” & “Red”.
  • Any part named accordingly.
  • GreenTrigger/RedTrigger fire on opposing carrier touch.

TeamSpecific

  • Folders: GRN & RED with Spawns, Checkpoints, SpawnZones, Barriers.
  • Spawns: respawn points per stage.
  • Checkpoints: optional respawns.
  • SpawnZones: class zones; opposing kills.
  • Barriers: block only opposite team.

Settings

  • Lighting, Terrain, and General settings.
  • Use the TC2 Map Plugin to automate Lighting & Terrain.
  • General toggles (AllowUnanchoredParts, Climbing) off unless needed.

GameMode

  • GameMode sets: "KOTH","AD","3CP","CTF","Arena","None".
  • KOTH: adds ActivateIn IntValue.
  • AD: adds GameTimer, FirstCap, SetupTimer.
  • 3CP: no extra values.
  • CTF: adds CritsOnCap, MaxTime, PlayTo, TimeOnCap.
  • Arena: adds ActivateIn, PrepTime.
  • None: adds PrepTime (infinite timer; end via VIP).

REAL Testmap_Extra

  • An expanded test map that showcases hidden TC2 map features, updated collision examples, and extra settings not obvious from the default plugin test maps.
  • Download the latest prefab: Testmap_Extra.rbxm. You can also load it from Roblox by ID 112829107151003.
  • Includes:
    • More accurate collision parts and the actual player collision box so you can see where players fit.
    • New map Settings examples (including seasonal / Halloween and seat “sitting” toggles).
    • Advanced hazards such as pumpkin bombs, conveyors, simple logic setups and other physics-based gimmicks.
  • Teleporters and doors in this test map demonstrate an important rule: set the Model’s PrimaryPart to the part you want used for hitbox / teleporter detection; without it, they will not work correctly.
  • Doors do not support a separate “delay before moving”, but the motion uses a tween: a longer MoveTime gives a longer ease-in/out. You can fake a delay by placing the door offset and increasing the Move distance.
